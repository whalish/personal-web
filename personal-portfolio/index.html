Bearmaps is an application used for mapping and routing that I created during my data structures course. It has similar functionality as Google maps and covers the East Bay area.

For rastering, a quadtree structure (each level containing images of different pixels per distance) was used allowing the user to both drag the map around and change the detail of the area shown by zooming in and out.

Data on all roads and pathways was deserialized from an XML file in order to construct a graph of all drivable roads. To draw routes, the A* algorithm was used with the straight line distance as a heuristic.

Locations were also read from the XML file and location finding was implemented by storing all the location names in a trie which allowed autocomplete. Searching for locations would reveal them on the map.

I worked in the Scalable Software Systems lab at the University of Florida. My work was in deep learning, mainly computer vision.
learn about the lab

A computer was trained to count the number of citrus in a picture, the images being supplied by the University's Department of Agriculture. The first rough method was to segment the image into many different subimages and classify each subimage as having citrus or not having citrus using the cifar-10 model of CNN, implemented in Keras run over Tensorflow. This was replaced by a more accurate method, written in Caffe, of using fast R-CNN to identify all regions of CNN and returning the total number of regions, and doing so quickly by using the feature map given by the first convolution to identify regions of interest.

The Deep Cloud project is a web app in development to apply all models created from research, including citrus counting. The website provides friendly UI such that users can train their own models or use pretrained models without knowledge of deep learning. The subsequent models are saved and stored on our internal server so that the user can access them later to run inference. As part of the deep learning subteam, I wrote python scripts to load data, build the model, and export the model architecture and weights in HDF5 and protobuff formats to our server. I also connected with the web team and worked on modifying callback functions in order to send ETA, loss, and validation loss values in JSON format during training.
